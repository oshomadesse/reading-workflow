🎯 核心メッセージ

  • 結論を先に示す
  • 根拠をMECEでグルーピング
  • ピラミッド構造で配置
  • 思考が明快になる
  • 相手の素早い納得と行動

📋 重要な引用

  • 読み手はまず答えを知りたがる
  • 上位は下位の要約でなければならない
  • 演繹か帰納のいずれかに統一
  • SCQA順で導入を設計する

🏗️ ピラミッド原則

  • 最重要メッセージを頂点に配置
  • 下位に関連アイデアをグルーピング
  • 階層化する表現・思考の枠組み
  • 上位は下位の要約が条件

⬆️ 結論先出し

  • 読み手の問いに対する答えを最初に提示
  • その後に理由・証拠で支える
  • 理解と意思決定を加速
  • トップダウン方式

📊 MECE

  • Mutually Exclusive（重複なく）
  • Collectively Exhaustive（漏れなく）
  • 項目を分解・整理する原則
  • 説得力と抜け漏れ防止に効果

📖 SCQA

  • Situation（状況）
  • Complication（複雑化）
  • Question（問い）
  • Answer（答え）

↔️ 水平方向の論理

  • 同じ階層のアイデア同士の並べ方
  • 演繹（一般→個別）で一貫
  • または帰納（事例→結論）で一貫
  • 混用は禁止

↕️ 垂直方向の要約

  • 上位メッセージは直下項目を総括
  • 下位は「なぜ」「だから」を支える
  • 要約規律を守る
  • 階層間の一貫性確保

🔄 SCQA導入フロー

S
状況（Situation）
現在の状況を共有し、前提を揃える
C
複雑化（Complication）
問題や課題を明確化し、緊急性を示す
Q
問い（Question）
解決すべき本質的な質問を設定
A
答え（Answer）
ピラミッドの頂点となる結論を提示

🔍 概念間の関係性

  • SCQAのQ→Aがピラミッドの頂点
  • 頂点Aを支える根拠をMECEでグルーピング
  • 各グループ内は演繹または帰納で一貫
  • 上位↔下位はWHY?/SO WHAT?で確認
  • 品質テストで論理と構造を検証

📚 構造と論理展開

  • 目的の明確化：結論先出しの重要性
  • 原則の提示：ピラミッド原則の定義
  • 導入設計：SCQAでの問題設定
  • グルーピング：関連性での束ね方
  • 品質テスト：MECEチェックと改善
