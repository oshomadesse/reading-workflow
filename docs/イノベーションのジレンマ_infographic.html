📚 イノベーションのジレンマ

優良企業ほど、既存顧客が求める高機能に集中するあまり、最初は安価で粗削りだが新しい価値を生む技術を見過ごし、やがて新参に市場を奪われる。解決には、既存事業と切り離した小さな組織で新しい基準の価値を学び、育てることが必要。

🔄 破壊的イノベーションの流れ

📈 主流顧客の要望重視 → 持続的改良に集中
⚡ 性能過剰が進行 + 無視された顧客層
🎯 下位/新市場の足場が生まれる
🚀 破壊的イノベーションが性能を伸ばす
🔄 新しい価値基準が主流化 → 逆転

🔥 破壊的イノベーション

最初は性能が低いが安価・簡単で新しい価値提案をもつ技術やビジネスが、無視された層から広がり、主流市場を席巻する現象
下からの逆転

📈 持続的イノベーション

既存顧客が重視する性能を既存製品の延長で高める改良。企業が通常得意とする進歩
延長線の改良

⚡ 性能過剰

製品の性能が顧客の実用需要を超え、余分な高機能が増える状態。ここに"十分で安い"新参がつけ入る
過ぎたる高機能

🌐 価値ネットワーク

顧客・競合・コスト構造が作る「何に価値があるか」の文脈。これが企業の判断基準を形作る
価値の文脈

🎯 下位市場/新市場の足場

過剰品質で不満の低価格志向層（下位）や未開拓の非顧客（新市場）に、簡素で安い提案で入り込む起点
入口の足がかり

🏗️ 資源・プロセス・価値基準（RPV）

人・資金などの資源、仕事の進め方、採算や優先順位の基準。組織の「できる/できない」を決める枠組み
組織の体質

🏢 自律組織（分離組織）

本体と別の目標・評価で動く小さなチーム。既存の採算基準に縛られず新しい価値を学べる
別室で育てる

🔬 探索学習

予測よりも小実験で顧客反応を学び、仮説を更新する進め方。未成熟市場に適する
小さく試す
