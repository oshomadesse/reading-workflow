📈 反脆弱性

不確実性・変動・ショックを避けるのではなく、それらから利益を得る設計をせよ。予測に依存せず、引き算・冗長・オプショナリティで脆さを減らし反脆弱性を育てる。

🎯 核心メッセージ

  • 不確実性から利益を得る設計を目指す
  • 予測精度向上より構造改善を重視
  • 引き算・冗長・オプションで脆弱性を削減
  • 非対称な賭けで上振れのみ捕捉
  • 小規模実験で情報を蓄積

⚖️ 三つの性質

  • 脆弱性：ショックで損なわれる 📉
  • 頑健性：衝撃に耐えるが成長しない ⚡
  • 反脆弱性：ストレスから利益を得る 📈
  • 目標は反脆弱なシステム構築
  • 変動性を味方につける設計思想

🔧 実装戦略

  • バーベル戦略：安全＋少額冒険
  • ヴィア・ネガティヴァ：害を引く
  • 冗長性：余裕を保険として活用
  • オプショナリティ：選択肢を多数保持
  • スキン・イン・ザ・ゲーム：身銭を切る

🚫 避けるべきもの

  • 医原病：過剰な介入による害
  • 予測依存：不確実性の過小評価
  • 破滅リスク：一発で終わる賭け
  • 複雑系への強引な介入
  • 利害の非対称：他人事の意思決定
